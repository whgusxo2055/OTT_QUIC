## 1. 개발 개요

### 1.1 개발 방법론

- **애자일 스크럼** 기반 개발
- 2주 단위 스프린트 (총 12개 스프린트)
- 각 스프린트 종료 시 동작하는 소프트웨어 산출

### 1.2 개발 단계

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                           6개월 개발 로드맵                                   │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  Phase 1         Phase 2         Phase 3         Phase 4         Phase 5   │
│  환경 구축        핵심 인프라       QUIC 구현       기능 개발        통합/배포  │
│  ─────────       ──────────      ──────────      ──────────      ────────  │
│  2주             6주             6주             8주             2주       │
│                                                                             │
│  [Sprint 1]     [Sprint 2-4]    [Sprint 5-7]    [Sprint 8-11]   [Sprint 12]│
│                                                                             │
│  • 환경 설정     • DB 설계        • UDP 소켓      • 인증           • 통합     │
│  • Docker       • WebSocket     • QUIC 프로토콜  • 스트리밍        • 테스트   │

└─────────────────────────────────────────────────────────────────────────────┘
```

### 1.3 개발 우선순위 원칙

1. **인프라 우선**: 통신 기반 → 기능 구현
2. **점진적 통합**: 각 모듈 독립 개발 후 통합
3. **QUIC 집중**: 핵심 학습 목표인 QUIC 구현에 충분한 시간 배정
4. **테스트 주도**: 각 스프린트마다 테스트 코드 작성

**QUIC 패킷 구조 (단순화)**:
```c
struct quic_packet {
    uint8_t  flags;           // 패킷 타입 플래그
    uint64_t connection_id;   // 연결 ID
    uint32_t packet_number;   // 패킷 번호
    uint32_t stream_id;       // 스트림 ID
    uint32_t offset;          // 데이터 오프셋
    uint32_t length;          // 페이로드 길이
    uint8_t  payload[];       // 가변 길이 데이터
};
```
